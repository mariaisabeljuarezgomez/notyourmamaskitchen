18. Undo History

The undo system lets you reverse mistakes one step at a time. The editor remembers your last 30 actions in its history stack.

What Undo Covers

  • Text content changes (typing, deleting characters)
  • Font family, font size, color, bold/italic changes
  • Letter spacing and line height changes
  • Element moves (drag to new position)
  • Element additions (Add Text, Add Rect, image placement)
  • Element deletions
  • Visibility toggles
  • Lock toggles
  • Shape fill/border style changes
  • Opacity changes
  • Rotation changes
  • Layer order changes (bring forward, send back)

What Undo Does NOT Cover

  • Reset to Original — this is an intentional full restore, not undoable
  • Load Session — replaces the entire state from the server
  • Changes made before the 30-step history limit
Shortcut: Press Ctrl + Z on Windows/Linux, or Cmd + Z on Mac, to undo without clicking the button. Works at any time while the editor is open.

23. Reset to Original

The 🔄 Reset to Original button restores your menu to its original factory template, removing all customizations. It is a complete reset — not a one-step undo.

How Reset Works

  1. Click 🔄 Reset to Original in the drawer.
  2. A confirmation dialog appears asking you to confirm the action.
  3. If you confirm, all custom text content, positions, styles, added elements, and uploaded image placements are cleared.
  4. The menu returns to the original template layout.
Warning: Reset to Original CANNOT be reversed with the Undo button. Before resetting, consider using 💾 Save Session to create a server backup of your current work. If you regret the reset, you can use 📂 Load Session to restore the last saved version.

When to Use Reset

Use Reset only when you want to start completely from scratch — for example, creating a seasonal menu from a clean template, or undoing extensive edits that went in the wrong direction. For single-step mistakes, use ↺ Undo Last Change instead.

You can start editing on one device and continue on another. The syncing is powered by the server-based Save/Load system.

How to Continue Work on Another Device

26. Troubleshooting

I cannot select or click any element

The layout may be Locked. Click the 🔒 Layout Locked button in the top header to switch to 🔓 Unlocked mode. Also confirm the element is not individually locked — check the Layers Panel and look for the 🔒 icon next to it.

I cannot move an element after selecting it

Either the global layout is Locked (header button), or the element itself is individually locked (LAYER tab or Layers Panel). Unlock both if needed. Also confirm you are clicking to select first, then dragging — do not drag without first confirming selection.

My text will not go into editing mode

Double-click (or double-tap on mobile) on the text element. A single click only selects it — you need a second click/tap to enter the text editor and place the cursor.

The Selection Bar is not visible

Make sure you have actually selected an element (single click). The bar only appears when something is selected. On mobile, scroll up on the canvas or close the drawer to make more room for the bar to appear.

I cannot see all the controls in the Selection Bar on my phone

The bar scrolls horizontally on mobile. Swipe left or right directly on the bar to see all tabs and buttons. The three tabs (LAYER, DESIGN, ARRANGE) and the duplicate and delete buttons should all be accessible by scrolling.

My changes are not showing on another device

You must click 💾 Save Session on the first device before using 📂 Load Session on the second. The editor does not sync automatically — only a manual Save sends the data to the server.

The background image looks blurry while editing

This is intentional. The editor uses a compressed preview version of the background for speed. Your exported PNG always uses the original full-resolution background and will be completely crisp and print-ready.

My export file is not downloading

Check your browser\'s download settings and make sure it allows downloads from the editor\'s site. On mobile, check that your browser has permission to save files. Try a different browser if the issue persists. Also check that you have enough storage space on your device.

The editor seems frozen or unresponsive

Save your work first (if possible), then click ↺ RELOAD in the top header to do a full page refresh. After reloading, click 📂 Load Session to restore your last saved version.

I accidentally deleted the wrong element

Immediately click ↺ Undo Last Change in the top header (or press Ctrl+Z). The deleted element will be restored as long as you have not exceeded the 30-step undo history.

An element ended up behind everything and I cannot reach it

Use the Layers Panel (in the drawer) to find the element by name, then click its row to select it. Once selected, go to the ARRANGE tab and click Bring to Front.

27. Best Practices

  • Save frequently. After finishing each menu section, after important changes, and always before closing the browser.
  • Keep the layout Locked by default. Only unlock when actively moving something. This is the single most effective habit for preventing accidents on both desktop and mobile.
  • Use Undo immediately when you notice a mistake — before doing anything else. The undo history clears older steps as you make new changes.
  • Zoom in on mobile before tapping small text like prices or fine-print descriptions. A zoomed-in tap is far more precise.
  • Name your elements meaningfully in the LAYER tab (e.g., "appetizer-section-title", "main-price-2"). This makes the Layers Panel much easier to navigate on complex menus.
  • Use the Layers Panel to select elements that are hard to click directly, hidden, or behind other elements.
  • Test your export before sending to print. Open the downloaded PNG file, zoom in to 100%, and verify every text element is correct and crisp.
  • Lock background elements. Any element that forms the permanent structure of your menu (borders, the background image, permanent logos) should be set to Locked so it can never be accidentally moved.
  • Use the ARRANGE tab for pixel-perfect alignment. If two prices need to be at exactly the same left edge, select each one and type the same X value.
  • Before resetting, always Save first so you can Load the last version if you change your mind.
